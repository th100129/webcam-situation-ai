# 웹캠 기반 집중도 감지 및 행동 미션 시스템

## 1. 프로젝트 개요

### 프로젝트명

웹캠 기반 집중도 감지 및 행동 미션 시스템

### 프로젝트 한줄 소개

웹캠 영상에서 얼굴, 눈, 손, 자세, 객체를 실시간으로 분석하여 사용자의 집중 흐림 상태를 감지하고, 집중력이 저하되었을 때 랜덤 행동 미션과 피드백을 제공하는 컴퓨터 비전 기반 집중 관리 프로그램이다.

### 개발 목적

학습이나 업무 중 사용자는 졸음, 스마트폰 확인, 주변 시선 분산, 자세 흐트러짐 등으로 집중력이 낮아질 수 있다. 단순 타이머나 알림은 사용자의 실제 상태를 반영하기 어렵다는 한계가 있다.

본 프로젝트는 웹캠 기반 비전 기술을 활용하여 사용자의 얼굴 방향, 눈 감김, 자세 변화, 손과 얼굴의 거리 등을 실시간으로 분석하고, 집중 흐림이 일정 시간 지속될 경우 경고음과 행동 미션을 제공하도록 구현하였다. 사용자는 미션을 수행해야 경고 상태를 해제할 수 있으며, 이를 통해 단순 알림보다 능동적인 집중 환기 경험을 제공하는 것을 목표로 하였다.

## 2. 주요 기능

### 2-1. 얼굴 및 시선 상태 분석

얼굴 랜드마크를 기반으로 얼굴이 정면을 향하고 있는지, 눈을 감고 있는지, 고개를 숙이고 있는지를 분석한다.

* 얼굴 정면, 좌측, 우측 방향 판단
* 눈 감김 상태 판단
* 눈 감김이 5초 이상 지속되는 경우 집중 흐림으로 판단
* 얼굴이 카메라 화면에서 일정 시간 사라지는 경우 감지
* 고개를 아래로 숙인 상태가 지속되는 경우 감지

### 2-2. 손 및 손가락 제스처 인식

손 랜드마크를 통해 손가락 개수와 손 모양을 분석하고, 미션 수행 여부를 판단한다.

* 손바닥
* 주먹
* 엄지척
* 브이
* 검지 가리키기
* 양손 들기
* 손가락 개수 추정
* 손과 얼굴의 거리 분석

### 2-3. 자세 변화 감지

상체 자세와 어깨 위치 변화를 활용하여 사용자의 자세가 많이 흔들리거나 기울어지는 상태를 감지한다.

* 어깨 기울기 분석
* 상체 중심 이동량 분석
* 자세 흔들림 상태 감지
* 일정 시간 이상 자세가 불안정한 경우 집중 흐림 신호로 반영

### 2-4. 객체 인식

YOLO 기반 객체 인식을 적용하여 웹캠 화면 속 사람 및 주요 객체를 감지한다.

* 사람 객체 감지
* 객체 등장 및 유지 시간 추적
* 여러 명의 사람이 감지되는 상황 확인
* 객체 상태 변화 시에만 로그 출력

### 2-5. 집중도 분석

여러 감지 결과를 하나의 집중도 판단 로직으로 통합하였다.

집중 흐림 판단에 활용되는 주요 행동은 다음과 같다.

| 집중 흐림 행동 | 판단 내용                           |
| -------- | ------------------------------- |
| 눈 감김     | 눈을 5초 이상 감고 있는 상태               |
| 얼굴 미감지   | 얼굴이 감지되던 상태에서 일정 시간 화면에서 사라진 경우 |
| 정면 이탈    | 얼굴을 좌우로 돌린 상태가 지속되는 경우          |
| 고개 숙임    | 아래를 보는 상태가 지속되는 경우              |
| 손-얼굴 접촉  | 손이 얼굴 주변에 오래 머무는 경우             |
| 자세 흔들림   | 어깨 기울기 또는 몸 중심 움직임이 큰 경우        |

집중 흐림 상태는 단일 행동만으로 즉시 확정하지 않고, 일정 시간 이상 지속되는지 확인하여 일시적인 움직임으로 인한 오탐을 줄이도록 구성하였다.

## 3. 집중 흐림 발생 시 동작 방식

집중 흐림이 감지되면 다음과 같은 순서로 동작한다.

1. 집중 흐림 원인을 확인한다.
2. 경고음이 재생된다.
3. 원인에 맞는 랜덤 행동 미션이 화면에 표시된다.
4. 사용자가 미션을 성공할 때까지 경고 상태를 유지한다.
5. 미션 성공 시 경고음을 중지한다.
6. 성공 문구와 파티클 효과를 표시한다.
7. 짧은 휴식 시간 후 다시 집중도 감지를 시작한다.

---

## 4. 랜덤 행동 미션

집중 흐림이 감지되면 사용자는 랜덤으로 제시되는 행동 미션을 수행해야 한다.

| 미션 종류    | 성공 조건               |
| -------- | ------------------- |
| 손바닥 보여주기 | 손바닥을 카메라에 일정 시간 유지  |
| 주먹 쥐기    | 주먹 상태를 일정 시간 유지     |
| 엄지척 하기   | 엄지손가락을 위로 든 상태 유지   |
| 브이 하기    | 검지와 중지를 편 브이 제스처 유지 |
| 검지 가리키기  | 검지만 편 상태 유지         |
| 정면 바라보기  | 정면을 보고 눈을 뜬 상태 유지   |
| 가만히 있기   | 자세 흔들림 없이 일정 시간 유지  |
| 양손 들기    | 양손을 어깨보다 위로 든 상태 유지 |

미션은 집중 흐림 원인에 따라 일부 후보군이 다르게 구성되며, 동일한 상황에서도 반복적으로 같은 행동만 제시되지 않도록 랜덤 방식으로 선택된다.

---

## 5. 미션 성공 피드백

미션 성공 시 사용자가 즉시 성공 여부를 인지할 수 있도록 시각적 피드백을 제공하였다.

* 미션 성공 문구 출력
* 미션 완료 시간 표시
* 화면 상단 성공 배너 출력
* 색종이 및 별 모양 파티클 효과
* 짧은 휴식 안내 문구 출력
* 경고음 중지

예시 문구는 다음과 같다.

> 좋아요! 4.4초 만에 미션을 완료했어요. 다시 집중해볼까요?

> 미션 성공! 잠시 쉬어갈게요.

## 8. 구현 과정

### 8-1. 웹캠 기반 객체 및 손 인식 구현

프로젝트 초기에는 YOLO 기반 객체 인식과 MediaPipe Hands를 적용하여 웹캠 화면에서 사람과 손을 감지하였다. 손가락 개수와 손바닥, 주먹 등의 기본 손 모양을 인식하도록 구현하였다.

### 8-2. 얼굴 랜드마크 및 눈 감김 분석 추가

MediaPipe Face Landmarker를 적용하여 얼굴 랜드마크를 추출하고, 양쪽 눈의 랜드마크 거리를 활용한 EAR(Eye Aspect Ratio) 방식으로 눈 감김 상태를 분석하였다.

눈이 일시적으로 깜빡이는 상황은 집중 흐림으로 판단하지 않도록 하고, 눈 감김이 5초 이상 지속되는 경우에만 집중 저하로 판단하도록 구성하였다.

### 8-3. 집중도 판단 로직 통합

얼굴 방향, 고개 숙임, 얼굴 미감지, 손-얼굴 접촉, 자세 흔들림, 눈 감김 정보를 통합하여 집중 흐림 점수를 계산하도록 구현하였다.

단순히 한 프레임에서 감지된 결과만 활용하지 않고, 행동이 일정 시간 이상 유지될 때만 집중 흐림 상태로 확정하여 안정성을 높이고자 하였다.

### 8-4. 미션 기반 피드백 기능 구현

집중 흐림이 감지되면 경고음과 함께 랜덤 행동 미션이 실행되도록 구현하였다. 사용자가 손 제스처, 얼굴 방향, 자세 유지 등의 행동을 수행하면 미션 성공으로 처리되며, 경고음이 중지되고 성공 피드백이 표시된다.

### 8-5. 한글 UI 및 파티클 효과 적용

OpenCV의 기본 텍스트 출력 기능은 한글 표현이 제한적이므로, Pillow와 Windows 한글 폰트를 사용하여 미션 문구와 성공 메시지가 깨지지 않도록 처리하였다.

또한 미션 성공 시 색종이와 별 모양의 파티클 효과를 출력하여 사용자가 성공 여부를 직관적으로 확인할 수 있도록 구성하였다.

## 10. 한계점

### 10-1. 처리 속도 문제

YOLO 객체 인식, 얼굴 랜드마크, 손 랜드마크, 자세 랜드마크를 동시에 실행하기 때문에 노트북 환경에서는 FPS가 낮아질 수 있다. 실제 테스트 환경에서는 약 6~10FPS 수준으로 확인되었으며, 실시간성 향상을 위한 최적화가 필요하다.

### 10-2. 조명 및 카메라 각도 영향

얼굴과 손 인식은 조명, 역광, 안경 반사, 카메라 각도, 사용자와 카메라 사이의 거리 등에 영향을 받을 수 있다. 특히 어두운 환경이나 얼굴이 일부 가려진 환경에서는 인식 정확도가 낮아질 수 있다.

### 10-3. 오탐 가능성

손이 얼굴 근처에 있는 행동은 턱 괴기나 집중 흐림 행동으로 판단될 수 있지만, 안경을 잠깐 고치거나 머리카락을 정리하는 행동도 손-얼굴 접촉으로 감지될 가능성이 있다.

### 10-4. 개인별 차이

눈 크기, 얼굴 형태, 자세 습관이 사용자마다 다르기 때문에 EAR 임계값, 고개 숙임 기준, 자세 흔들림 기준을 모든 사용자에게 동일하게 적용하기 어렵다.

## 11. 개선 방향

### 11-1. 사용자별 임계값 보정

프로그램 시작 전 사용자의 평상시 눈 크기, 얼굴 방향, 자세를 측정하여 개인별 기준값을 설정하는 기능을 추가할 수 있다.

### 11-2. 성능 최적화

YOLO 객체 인식 실행 주기를 줄이거나, 프레임마다 모든 모델을 실행하지 않고 필요한 모델만 선택적으로 실행하여 FPS를 개선할 수 있다.

### 11-3. 집중도 점수 시각화

시간에 따른 집중도 변화를 그래프나 타임라인 형태로 제공하여 사용자가 자신의 집중 패턴을 확인할 수 있도록 확장할 수 있다.

### 11-4. 세션 리포트 고도화

학습 종료 후 집중 흐림 발생 횟수, 원인별 발생 비율, 미션 성공률, 집중 유지 시간 등을 자동으로 정리한 리포트를 제공할 수 있다.

### 11-5. 개인화 미션 추천

사용자가 자주 실패하는 미션이나 자주 발생하는 집중 흐림 원인에 따라 미션 난이도와 종류를 개인화할 수 있다.

## 12. 기대 효과

본 프로젝트는 단순히 사용자를 감시하는 시스템이 아니라, 사용자의 집중 흐림 상태를 감지한 뒤 즉시 행동 변화를 유도하는 인터랙티브 집중 관리 시스템이다.

특히 웹캠, 얼굴 랜드마크, 손 제스처, 자세 분석, 객체 인식, 미션 피드백을 하나의 흐름으로 결합하여 사용자가 실제로 반응하고 집중 상태를 회복하도록 설계했다는 점에 의미가 있다.

향후 개인별 집중 패턴 분석, 장기 학습 리포트, AI 기반 피드백 문구 생성, 개인화 미션 추천 기능을 추가한다면 학습 보조 서비스나 재택근무 집중 관리 서비스로 확장할 수 있을 것으로 기대된다.
