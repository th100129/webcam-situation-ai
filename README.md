# Focus Trigger

## 웹캠 기반 집중 흐름 감지 및 LLM 미션 피드백 시스템

## 1. 프로젝트 개요

**Focus Trigger**는 웹캠을 통해 사용자의 얼굴 방향, 손동작, 자세 변화, 움직임 패턴을 실시간으로 분석하고, 집중 흐름이 흐려진 것으로 보이는 순간 LLM이 상황에 맞는 랜덤 미션과 피드백 문구를 생성하는 인터랙티브 컴퓨터 비전 시스템이다.

사용자가 정면을 바라보고 자세가 안정적인 상태에서는 별도의 자극을 주지 않는다. 하지만 고개 돌림, 얼굴 이탈, 손-얼굴 접촉, 자세 흐트러짐, 무반응 등의 행동이 일정 시간 이상 감지되면 집중 흐림 가능성이 있다고 판단한다.

이후 시스템은 손바닥 보이기, 엄지척 하기, 브이 하기, 정면 바라보기, 움직이지 않기 등의 랜덤 미션을 제공한다. 사용자의 미션 수행 여부는 CV 모듈이 판정하고, 미션 성공 시에는 이모지 파티클 효과를 출력한다. LLM은 감지 상황과 미션 결과를 바탕으로 자연스러운 미션 문구와 피드백을 생성한다.

## 2. 프로젝트 목표

- 웹캠 기반 사용자 상태 분석
- 얼굴 방향, 손동작, 자세 변화, 움직임 패턴 분석
- 집중 흐림 가능성 점수화
- 집중 흐림 감지 시 랜덤 미션 제공
- 손동작 및 자세 기반 미션 성공 여부 판정
- LLM 기반 상황 맞춤 미션 문구 생성
- LLM 기반 성공/실패 피드백 생성
- 미션 성공 시 이모지 파티클 효과 출력
- 반응 시간, 성공률, 이벤트 로그 저장

## 4. 시스템 흐름

1. 웹캠으로 사용자 영상을 입력받는다.
2. MediaPipe를 이용해 얼굴, 손, 자세 랜드마크를 추출한다.
3. 얼굴 방향, 고개 숙임, 손-얼굴 접촉, 자세 흔들림 등을 분석한다.
4. 집중 흐림 가능성 점수를 계산한다.
5. 점수가 기준 이하로 떨어지면 미션 트리거를 발생시킨다.
6. 시스템이 현재 상황에 맞는 미션 타입을 선택한다.
7. LLM이 미션 문구를 자연어로 생성한다.
8. 사용자가 미션을 수행한다.
9. CV 모듈이 미션 성공 여부를 판정한다.
10. 미션 성공 시 이모지 파티클 효과를 출력한다.
11. LLM이 성공 또는 실패 피드백 문구를 생성한다.
12. 반응 시간, 성공 여부, 감지 이유를 로그로 저장한다.

## 6. 중요한 설계 방향

본 프로젝트에서 LLM이 영상을 직접 보고 집중도를 판단하는 구조는 아니다.

집중 흐림 여부와 미션 성공 여부는 CV 모듈이 담당한다. LLM은 CV가 분석한 결과를 입력받아 사용자에게 보여줄 문구를 생성한다.

구조는 다음과 같다.

**웹캠 영상 → CV 분석 → 구조화된 상태 데이터 → LLM 문구 생성 → 사용자 피드백**

이 방식은 영상 원본을 LLM에 직접 전달하지 않기 때문에 처리 속도와 개인정보 측면에서 더 안전하고, CV 파트의 역할도 명확하게 유지할 수 있다.

## 7. 집중 흐림 가능성 판단 기준

| 관찰 요소 | 판단 기준 | 의미 |
| --- | --- | --- |
| 얼굴 방향 | 일정 시간 이상 정면이 아닌 방향을 바라봄 | 화면 이탈 가능성 |
| 얼굴 미감지 | 얼굴이 일정 시간 이상 화면에서 사라짐 | 자리 이탈 가능성 |
| 고개 숙임 | 얼굴 중심점이 아래로 이동하거나 머리 각도가 낮아짐 | 졸림 또는 집중 저하 가능성 |
| 손-얼굴 접촉 | 손 랜드마크가 얼굴 영역 근처에 오래 머묾 | 피로 또는 산만 행동 가능성 |
| 자세 흔들림 | 어깨 좌표, 얼굴 중심점 변화가 잦음 | 집중 흐름 저하 가능성 |
| 무반응 | 지나치게 오랜 시간 움직임이 거의 없음 | 졸림 또는 무반응 가능성 |

## 8. 집중 흐림 점수 계산 예시

기본 집중 점수는 100점으로 설정하고, 특정 행동이 감지될 때마다 점수를 차감한다. 점수가 기준 이하로 내려가면 집중 흐림 가능성이 있다고 판단하고 랜덤 미션을 발생시킨다.

| 조건 | 점수 변화 |
| --- | --- |
| 정면 응시 유지 | +1 |
| 고개 돌림 3초 이상 | -15 |
| 얼굴 미감지 2초 이상 | -25 |
| 고개 숙임 3초 이상 | -15 |
| 손-얼굴 접촉 3초 이상 | -10 |
| 자세 흔들림 증가 | -10 |
| 무반응 10초 이상 | -10 |
| 미션 성공 | +15 |
| 미션 실패 | -10 |

## 9. 랜덤 미션 종류

| 미션 | 판정 방식 |
| --- | --- |
| 손바닥 보여주기 | 다섯 손가락이 펼쳐졌는지 확인 |
| 주먹 쥐기 | 손가락이 모두 접힌 상태인지 확인 |
| 엄지척 하기 | 엄지만 펼쳐지고 나머지 손가락이 접혔는지 확인 |
| 브이 하기 | 검지와 중지만 펼쳐졌는지 확인 |
| 정면 3초 바라보기 | 얼굴 방향이 정면으로 일정 시간 유지되는지 확인 |
| 움직이지 않기 | 프레임 간 움직임 변화량이 기준 이하인지 확인 |
| 양손 들기 | 양쪽 손목 좌표가 어깨보다 위에 있는지 확인 |
| 고개 들기 | 얼굴 중심점이 정상 위치로 복귀했는지 확인 |

## 10. LLM 문구 생성 예시

| 감지 상황 | 미션 타입 | LLM 생성 문구 예시 |
| --- | --- | --- |
| 고개 돌림 | 엄지척 | 집중력이 잠깐 화면 밖으로 나간 것 같아요. 3초 안에 엄지척으로 복귀 신호를 보내주세요. |
| 얼굴 미감지 | 손바닥 | 잠깐 자리를 벗어난 것 같아요. 돌아왔다면 손바닥을 보여주세요. |

| 고개 숙임 | 고개 들기 | 고개가 조금 내려갔어요. 화면을 보고 고개를 들어주세요. |

## 11. LLM 피드백 예시

| 성공 행동 | 출력 효과 |
| --- | --- |

| 주먹 | 폭발 이펙트 |

## 16. CV 심화 요소

### 랜드마크 기반 행동 분석

단순히 사람을 감지하는 것이 아니라 얼굴, 손, 상체의 랜드마크 좌표를 기반으로 사용자의 행동을 분석한다.

분석 대상은 다음과 같다.

- 얼굴 중심점
- 눈, 코, 턱 위치
- 손가락 끝 좌표
- 손목 좌표
- 어깨 좌표
- 팔꿈치 좌표

### 시간 기반 상태 판정

한 프레임에서만 감지된 행동은 오인식일 수 있으므로, 일정 시간 이상 유지된 경우에만 상태로 인정한다.

예시는 다음과 같다.

- 고개 돌림 3초 이상 유지 시 집중 흐림 가능성으로 판단
- 얼굴 미감지 2초 이상 유지 시 자리 이탈 가능성으로 판단
- 손-얼굴 접촉 3초 이상 유지 시 산만 행동 가능성으로 판단
- 동일 손동작이 일정 프레임 이상 유지될 때만 미션 성공 처리

### 움직임 변화량 분석

프레임 간 얼굴 중심점, 어깨 좌표, 손목 좌표의 변화량을 계산하여 자세 흔들림이나 무반응 상태를 판단한다.

움직임 점수는 다음 요소를 조합하여 계산한다.

- 얼굴 이동량
- 어깨 이동량
- 손 이동량
- 전체 프레임 변화량

### 제스처 미션 판정

| 제스처 | 판정 기준 |
| --- | --- |
| 손바닥 | 다섯 손가락이 모두 펼쳐진 상태 |

| 엄지척 | 엄지만 위쪽으로 펼쳐지고 나머지 손가락은 접힌 상태 |

### 오인식 방지 로직

웹캠 기반 인식은 프레임마다 결과가 흔들릴 수 있으므로 안정화 로직을 적용한다.

- 최근 N프레임 결과의 다수결
- 동일 상태가 일정 시간 이상 유지될 때만 인정
- 미션 성공 후 쿨타임 적용
- 얼굴 또는 손 감지 신뢰도가 낮으면 판정 보류

## 17. LLM 입력 데이터 예시

LLM에는 영상 원본을 전달하지 않고, CV 모듈이 분석한 구조화 데이터를 전달한다.

| 항목 | 예시 |
| --- | --- |
| focus_score | 58 |
| trigger_reason | head_turned |
| duration | 4.2초 |
| selected_mission | thumb_up |
| recent_success_rate | 70% |
| last_reaction_time | 1.3초 |
| difficulty | medium |

LLM은 위 정보를 바탕으로 미션 문구, 성공/실패 피드백, 세션 요약을 생성한다.

## 22. 차별점

기존의 일반적인 웹캠 기반 CV 프로젝트는 웹캠 입력 후 사람 또는 손을 감지하고 라벨을 출력하는 구조가 많다.

본 프로젝트는 웹캠 입력 후 얼굴, 손, 자세 상태를 분석하고, 집중 흐림 가능성을 추정한 뒤 랜덤 미션을 제시한다. 이후 사용자의 반응을 다시 컴퓨터 비전으로 판정하고, 미션 성공 시 이모지 파티클 피드백을 제공한다.

또한 CV 분석 결과를 LLM에 전달하여 상황에 맞는 미션 문구, 성공/실패 피드백, 세션 요약 리포트를 생성한다.

즉, 단순 감지 결과 출력이 아니라 사용자 상태 분석, 행동 트리거, CV 기반 미션 판정, LLM 기반 자연어 피드백, 이모지 시각 효과까지 연결된 인터랙티브 AI 시스템이라는 점에서 차별성이 있다.
